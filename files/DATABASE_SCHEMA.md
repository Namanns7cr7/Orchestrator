# Database Schema

Implementation note: SQLite is sufficient for the hackathon scope (see `TRD.md` Section 4); the schema below is written in SQL-ish form but maps directly to plain CSV/JSON files if preferred.

## `claims`
| Column | Type | Notes |
|---|---|---|
| `claim_id` | TEXT, PRIMARY KEY | e.g. `CLM-1042` |
| `user_id` | TEXT, FOREIGN KEY → `user_history.user_id` | |
| `object_type` | TEXT | one of `car`, `laptop`, `package` |
| `claimed_part` | TEXT | e.g. `rear_bumper` |
| `claimed_damage` | TEXT | semicolon-separated, e.g. `dent;paint_scrape` |
| `conversation` | TEXT | raw claim conversation / chat transcript |
| `created_at` | TIMESTAMP | |

## `images`
| Column | Type | Notes |
|---|---|---|
| `image_id` | TEXT, PRIMARY KEY | e.g. `IMG-1042-2` |
| `claim_id` | TEXT, FOREIGN KEY → `claims.claim_id` | |
| `path` | TEXT | local file path |

## `image_analysis`
One row per `(image_id)`, written by the Vision Analysis Agent + Authenticity Agent.
| Column | Type | Notes |
|---|---|---|
| `image_id` | TEXT, PRIMARY KEY, FOREIGN KEY → `images.image_id` | |
| `detected_object` | TEXT | |
| `detected_part` | TEXT | |
| `damage_type` | TEXT | semicolon-separated detected damage types |
| `severity` | TEXT | `none` \| `minor` \| `moderate` \| `severe` |
| `quality_score` | REAL | 0–1, derived from authenticity checks (1.0 = no flags) |
| `authenticity_flags` | TEXT | semicolon-separated flag_types, empty if none |
| `vision_confidence` | REAL | 0–1, from Vision Analysis Agent |

## `user_history`
| Column | Type | Notes |
|---|---|---|
| `user_id` | TEXT, PRIMARY KEY | |
| `prior_claims_count` | INTEGER | |
| `prior_claims_same_object_type` | INTEGER | |
| `prior_claims_same_part` | INTEGER | |
| `account_age_days` | INTEGER | |
| `prior_fraud_flags` | INTEGER | |

## `evidence_requirements`
One row per checklist item per claim, written by the Evidence Requirement Agent and updated by the Evidence Matching Agent.
| Column | Type | Notes |
|---|---|---|
| `requirement_id` | TEXT, PRIMARY KEY | e.g. `CLM-1042-R1` |
| `claim_id` | TEXT, FOREIGN KEY → `claims.claim_id` | |
| `evidence_type` | TEXT | e.g. `damage_visible_on_claimed_part` |
| `criticality` | TEXT | `critical` \| `supporting` |
| `satisfied` | BOOLEAN | |
| `satisfied_by_image_id` | TEXT, NULLABLE, FOREIGN KEY → `images.image_id` | |

## `agent_outputs`
Full audit trail — one row per agent invocation per claim (the raw reasoning trace referenced in `PRD.md` deliverables).
| Column | Type | Notes |
|---|---|---|
| `id` | INTEGER, PRIMARY KEY, AUTOINCREMENT | |
| `claim_id` | TEXT, FOREIGN KEY → `claims.claim_id` | |
| `agent_name` | TEXT | one of the 7 agents in `AGENT_ARCHITECTURE.md` |
| `output_json` | TEXT (JSON) | full raw output of that agent for that claim |
| `created_at` | TIMESTAMP | |

## `evidence_coverage`
One row per claim, written by the Evidence Matching Agent — separated from `final_decisions` so coverage can be inspected/evaluated independently of the final label.
| Column | Type | Notes |
|---|---|---|
| `claim_id` | TEXT, PRIMARY KEY, FOREIGN KEY → `claims.claim_id` | |
| `evidence_coverage_score` | REAL | 0–1, formula in `SYSTEM_DESIGN.md` Section 3 |
| `contradictions_found` | INTEGER | count of `contradicts` edges |
| `unsatisfied_critical_count` | INTEGER | |

## `final_decisions`
| Column | Type | Notes |
|---|---|---|
| `claim_id` | TEXT, PRIMARY KEY, FOREIGN KEY → `claims.claim_id` | |
| `decision` | TEXT | `SUPPORTED` \| `CONTRADICTED` \| `INSUFFICIENT_EVIDENCE` |
| `confidence` | REAL | 0–1, formula in `SYSTEM_DESIGN.md` Section 4 |
| `needs_manual_review` | BOOLEAN | |
| `justification` | TEXT | human-readable, generated by Final Judge Agent |
| `decided_at` | TIMESTAMP | |

## Entity-relationship summary
```
user_history (1) ──< (many) claims (1) ──< (many) images (1) ── (1) image_analysis
                                  │
                                  ├──< (many) evidence_requirements
                                  ├──< (many) agent_outputs
                                  ├── (1) evidence_coverage
                                  └── (1) final_decisions
```

## Sample rows (reference claim `CLM-1042`, see `PRD.md` Section 8)
```sql
-- claims
INSERT INTO claims VALUES ('CLM-1042','U-201','car','rear_bumper','dent;paint_scrape',
  'Someone backed into my rear bumper in the parking lot...', '2026-06-01T10:00:00Z');

-- final_decisions
INSERT INTO final_decisions VALUES ('CLM-1042','SUPPORTED',0.91,0,
  'Both submitted photos show a dent and paint scrape on the rear bumper consistent with the claim...',
  '2026-06-01T10:00:42Z');
```

## `output.csv` (flat export — what gets handed in as the deliverable)
Derived by joining `claims`, `evidence_coverage`, and `final_decisions`:
```
claim_id, decision, confidence, evidence_coverage_score, needs_manual_review, justification
CLM-1042, SUPPORTED, 0.91, 1.0, false, "Both submitted photos show a dent and paint scrape..."
```
